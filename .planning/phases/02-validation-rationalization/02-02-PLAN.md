<objective>
Relax two overly strict input validators so operators can use CIDR-format EPG names and /30-/31 subnet masks.

Purpose: Operators routinely name segments after the prefix they represent (e.g., '192.168.1.0/24') and deploy /30 point-to-point links. The current regex and subnet thresholds are not protecting any data invariant — they are blocking legitimate operator workflows. RFC 3021 defines /31 as valid for point-to-point links; NetBox has no character restrictions on VLAN names.

Output: input_validators.py regex updated; network_validators.py updated in two places (validate_subnet_mask upper bound and validate_network_broadcast_gateway minimum-address threshold with removal of redundant usable_hosts check).
</objective>
